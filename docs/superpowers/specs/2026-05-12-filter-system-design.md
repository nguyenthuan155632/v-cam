# V-Cam Filter / Preset System — Design

**Date:** 2026-05-12
**Status:** Approved, ready for implementation plan

## Problem

V-Cam ships with a non-functional filter system:

- All 24 `.cube` files in `app/src/main/assets/luts/` are **identity LUTs** (no-op). The live preview renderer applies them faithfully but the math collapses to passthrough — every filter looks identical.
- `CameraController.capture` writes the raw JPEG straight to MediaStore with **no filter applied**. Saved photo never matches the live preview.
- `PhotoPreviewScreen` fakes the filter look with a linear gradient overlay (`Filter.tintTop`/`tintBottom`), which is unrelated to the LUT pipeline. Cosmetic deception.
- Existing filter taxonomy (Cafe, Vintage, Clean, Warm, Cool, …) doesn't match the product brief; some names (`Polaroid`, `1978`) carry trademark risk.
- No system for adding new filters: `Filters.kt` is hand-edited, `.cube` files are placeholders, no story for sourcing real grades under the project's license rule.

## Goal

Production-ready filter system that:

1. Shows real, distinct filter looks in live preview.
2. Saves photos with the chosen filter baked in at full resolution and JPEG quality 100.
3. Maintains pixel parity between live preview and saved JPEG.
4. Lets the user re-apply or change the filter on the saved photo without quality loss.
5. Supports easy addition of new filters via Kotlin data + a Gradle bake task.
6. Ships with a curated catalog of ~30 filters across 6 categories (Food, Portrait, Film, Travel, Night, Mono), expandable to 50.
7. Contains zero third-party LUT files. All filter data and reference imagery is procedurally generated and owned by the project.

## Non-goals

- HDR / Night CameraX extensions (kept in capture pipeline scope; not enabled in v1).
- Tiled rendering for images larger than `GL_MAX_TEXTURE_SIZE` (downscale fallback only).
- User-authored custom filters.
- Filter favorites, reordering, or user catalog editing.
- Tap-to-focus / exposure compensation changes — existing code kept untouched.

## Architecture

Three layers, each independently testable:

```
┌─────────────────────────────────────────────────────────────┐
│  BUILD-TIME (Gradle task `bakeLuts`, JVM, no Android deps)  │
│  FilterRecipes (params) ─┐                                  │
│                          ├─▶ LutBaker → assets/luts/*.cube  │
│  FilterParams ───────────┘                                  │
│  ReferenceImageBaker ────▶ assets/thumbs/reference.png      │
└─────────────────────────────────────────────────────────────┘
                          │ (assets bundled in APK)
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  RUNTIME — color  (com.vcam.color)                          │
│  CubeLutParser (existing, unchanged)                        │
│  LutCache       (new, LRU keyed by filter id)               │
│  ThumbnailRenderer (new, CPU trilinear LUT on ref image)    │
│  OffscreenLutProcessor (new, EGL pbuffer + same shader)     │
└─────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  RUNTIME — features                                          │
│  CameraScreen / LutRenderer  — live preview (path unchanged)│
│  CameraController.capture     — calls OffscreenLutProcessor │
│  PhotoPreviewScreen           — re-applies via OffscreenLut │
│  FilterBrowser/Ribbon/Thumb   — uses ThumbnailRenderer      │
└─────────────────────────────────────────────────────────────┘
```

### Key invariants

- **One shader source of truth** (`LutShaders.kt`). Both live preview and offscreen capture re-render share the LUT lookup + intensity-lerp GLSL functions. Pixel parity guaranteed.
- **Procedural params own everything.** A filter is a `FilterParams` data class. `.cube` files are derived artifacts produced by `bakeLuts`. Adding a filter = add Kotlin val + run `./gradlew bakeLuts`.
- **No third-party LUTs in the repo.** Existing identity `.cube` files are deleted; all `.cube` files are generated.
- **License-clean reference image.** Procedural test pattern, generated by the same Gradle task.
- **id-based filter selection.** Filters identified by stable string id, not array index — catalog edits don't break saved settings.

## Filter data model

New package `com.vcam.color`. Replaces `data/Filter.kt`, `data/Filters.kt`, `data/Categories.kt`.

```kotlin
enum class FilterCategory(val displayName: String) {
    Food("Food"), Portrait("Portrait"), Film("Film"),
    Travel("Travel"), Night("Night"), Mono("Mono"),
}

data class FilterParams(
    val whiteBalance: WhiteBalance = WhiteBalance(),     // temp shift, tint shift
    val lift: Rgb = Rgb(0f, 0f, 0f),                     // shadow color shift  [-1..1]
    val gamma: Rgb = Rgb(1f, 1f, 1f),                    // midtone power        [0.1..3]
    val gain: Rgb = Rgb(1f, 1f, 1f),                     // highlight scale     [0..3]
    val saturation: Float = 1f,                          // 0=mono, 1=neutral
    val contrast: Float = 1f,                            // pivot 0.5
    val brightness: Float = 0f,                          // [-0.5..0.5]
    val channelMixer: ChannelMixer = ChannelMixer.identity(), // 3x3
    val toneCurve: ToneCurve = ToneCurve.linear(),       // control points
    val splitToning: SplitToning? = null,                // optional
)

data class Filter(
    val id: String,                  // stable, e.g. "food_fresh"
    val category: FilterCategory,
    val displayName: String,         // user-facing, e.g. "Fresh"
    val shortCode: String,           // ribbon code, e.g. "FD·1"
    val lutAsset: String,            // "luts/food_fresh.cube"
    val intensityCap: Float = 1f,    // upper bound (0..1) on intensity slider; enforced in
                                     // CameraViewModel.setIntensity / PhotoPreviewViewModel.setIntensity
)

object FilterCatalog {
    val all: List<Filter>            // ~30 filters, 6 categories
    fun byCategory(cat: FilterCategory): List<Filter>
    fun byId(id: String): Filter?
}
```

Procedural recipes (`FilterRecipes.all: Map<String, FilterParams>`) live in a **build-only** source set (`src/bakerMain/kotlin/...`) and never ship in the APK.

### Naming guard

Every `displayName` and `shortCode` reviewed for trademark. Generic names only: `Classic Cool`, `Soft Film`, `Warm Vintage`, `Noir`, etc. Internal recipe ids (`food_fresh`) are unconstrained.

### Categories and approximate distribution (~30 filters, room for ~50)

- **Food** (5): Fresh, Sweet, Warm Table, Creamy, Garden
- **Portrait** (5): Clean Bright, Soft Skin, Warm Portrait, Pink Tint, Studio Glow
- **Film** (6): Classic Cool, Soft Film, Warm Vintage, Faded Negative, Pushed Color, Muted Frame
- **Travel** (5): Summer Pop, Beach Bright, City Clear, Teal Orange, Golden Hour
- **Night** (4): City Night, Neon Soft, Low Light Warm, Moody Blue
- **Mono** (4): Soft B&W, High Contrast B&W, Noir, Warm Mono

Final names finalized during implementation.

## Build-time pipeline

### Module layout

```
app/
├── src/main/...                          # Android app code (shipped)
└── src/bakerMain/kotlin/com/vcam/baker/  # JVM-only (NOT shipped)
    ├── FilterRecipes.kt                  # ~30 procedural recipes
    ├── LutBaker.kt                       # FilterParams → CubeLut → .cube
    ├── ColorPipeline.kt                  # the math
    ├── ReferenceImageBaker.kt            # generates thumbs/reference.png
    └── Main.kt                           # Gradle entry point
```

### Color pipeline order (must match in baker and shader semantics)

```
input rgb ∈ [0,1]
  1. white balance     (temp shift; tint shift on G)
  2. lift / gamma / gain (per-channel)
  3. brightness        (additive)
  4. contrast          (pivot 0.5)
  5. channel mixer     (3x3 multiply)
  6. saturation        (lerp toward luma; luma = 0.299r + 0.587g + 0.114b)
  7. split toning      (optional; lerp toward shadow/highlight tint by luma)
  8. tone curve        (Catmull-Rom interp through control points)
  9. clamp [0,1]
output rgb → cube entry
```

Each step is a small pure function in `ColorPipeline.kt`, JVM-unit-testable.

### LutBaker

- Cube size **33** (industry-standard authoring resolution; `CubeLutParser` already supports any N).
- Cube ordering: **R fastest, then G, then B** — must match `CubeLutParser` (verified at `LutRenderer.uploadLut` lines 219–221) and existing `parseCubeLut`. This is a hard invariant captured in tests.
- Output: human-readable `.cube` file with `LUT_3D_SIZE 33`, `DOMAIN_MIN 0 0 0`, `DOMAIN_MAX 1 1 1`, then `33³ = 35937` lines of `r g b`.
- **Baked LUTs are always full strength.** Intensity is *not* baked into the LUT; it is applied at render time (live preview shader, offscreen processor) by lerping the LUT output against the original pixel. Thumbnails render at intensity=1.0 to show the filter's full character.

### Reference image baker

256×256 PNG with deliberate test patches:
- Horizontal grayscale ramp (shadows → highlights)
- Primary color patches (R, G, B)
- 4 skin-tone patches (ITU-R approximation)
- Saturation ramp
- White-balance reference patches (tungsten / daylight / shade)

Generated in JVM AWT (host-only); shipped to `assets/thumbs/reference.png`. License-clean.

### Gradle wiring

```kotlin
sourceSets {
    create("bakerMain") { kotlin.srcDirs("src/bakerMain/kotlin") }
}

tasks.register<JavaExec>("bakeLuts") {
    group = "vcam"
    description = "Generate procedural LUT .cube files and reference image into assets/."
    classpath = sourceSets["bakerMain"].runtimeClasspath
    mainClass.set("com.vcam.baker.MainKt")
    args(layout.projectDirectory.dir("src/main/assets").asFile.absolutePath)
    inputs.dir("src/bakerMain/kotlin")
    outputs.dir("src/main/assets/luts")
    outputs.file("src/main/assets/thumbs/reference.png")
}

tasks.named("preBuild") { dependsOn("bakeLuts") }
```

`bakeLuts` is up-to-date-checked: unchanged recipes → no work. Manual: `./gradlew bakeLuts`.

## Runtime — live preview path (largely unchanged)

`LutRenderer` and the existing GL pipeline stay. Once procedural `.cube` files replace identity placeholders, the live preview becomes the real grade automatically.

Single addition: **`LutCache`** (LRU, ~8 entries, ~1 MB) so rapid filter swaps don't re-parse the same `.cube` file from disk repeatedly.

`LutShaders.kt` refactored:

```
LUT_LOOKUP_GLSL    // shared trilinear LUT lookup (function string)
INTENSITY_LERP_GLSL// shared lerp(input, graded, intensity)
PREVIEW_FRAGMENT   // OES external sampler + above shared functions
OFFSCREEN_FRAGMENT // 2D sampler + above shared functions
VERTEX_SHADER      // shared
```

The **identical** lookup math in both shaders is what guarantees preview/capture parity.

## Runtime — capture pipeline rewrite

### Today's flow (broken)

`ImageCapture.takePicture(OutputFileOptions, OnImageSavedCallback)` writes raw JPEG straight to MediaStore. Filter never applied.

### New flow

`ImageCapture.takePicture(executor, OnImageCapturedCallback)` — JPEG bytes returned in memory. Unfiltered JPEG never hits disk in the success path.

```
shutter tap
   │
   ▼
ImageCapture.takePicture(executor, OnImageCapturedCallback)
   │  (CameraX still at MAXIMIZE_QUALITY, full sensor resolution)
   ▼
ImageProxy (JPEG bytes + EXIF + rotation)
   │
   ▼  post to OffscreenLutProcessor.handlerThread
   │
   1. Decode JPEG → Bitmap (ARGB_8888, full res)
   2. Apply EXIF rotation
   3. Bind to GL_TEXTURE_2D, render through LUT shader → FBO
   4. glReadPixels → ByteBuffer → Bitmap
   5. Encode JPEG quality 100 → ByteArray
   6. Write to MediaStore Pictures/VCam/VCam_<ts>.jpg
   7. If saveOriginal: also write VCam_<ts>_orig.jpg
   8. Restore EXIF on both files
   │
   ▼  callback on main executor
saved Uri → UI
```

### OffscreenLutProcessor

```kotlin
class OffscreenLutProcessor {
    private val handlerThread = HandlerThread("VCamLutProcessor").apply { start() }
    private val handler = Handler(handlerThread.looper)

    // Independent EGL pbuffer context — does not touch preview's GL context.
    private var eglDisplay: EGLDisplay
    private var eglContext: EGLContext
    private var eglSurface: EGLSurface
    private var program: Int
    // FBO + uniform locations cached after first use.

    fun process(source: Bitmap, lut: CubeLut, intensity: Float, callback: (Bitmap?) -> Unit)
    fun release()
}
```

- All GL calls on the processor's `HandlerThread`. Bitmap decode/encode also on that thread (no cross-thread bouncing).
- Result delivered on `mainExecutor`.
- One in-flight capture max; shutter UI disabled while processing.
- Lifecycle: lazily created on first capture, owned by `CameraController`, `release()`d in `onDispose`.
- Reused by `PhotoPreviewScreen` via app-level singleton (`VCamApplication.offscreenProcessor`).

### Save semantics (respects `UserSettings.saveOriginal`)

| `saveOriginal` | Filtered file | Original file |
|----------------|---------------|---------------|
| true (default) | `VCam_<ts>.jpg` | `VCam_<ts>_orig.jpg` |
| false          | `VCam_<ts>.jpg` | (none) |

Both written via `ContentResolver.openOutputStream` + `Bitmap.compress(JPEG, 100, ...)`.

EXIF copied from `ImageProxy` metadata onto both files. Tags preserved: `Orientation`, `DateTime`, `Make`, `Model`, `ExposureTime`, `ISOSpeedRatings`, `FNumber`, `FocalLength`, `WhiteBalance`, `GPS*`. Custom tag added: `UserComment = "V-Cam filter:<id> intensity:<n>"`.

**Optimization:** when intensity = 0% AND filter = identity → skip GPU re-render, write source JPEG bytes directly.

### Resolution guard

- Cache `GL_MAX_TEXTURE_SIZE` once.
- If `max(w, h) > GL_MAX_TEXTURE_SIZE`: downscale to fit, log warning. (Modern phones at 12MP = 4032×3024 fit in 4096; older ceiling devices get a downscale.)
- v2: tiled rendering. Out of scope here.

## Runtime — PhotoPreviewScreen rewrite

State change in `PhotoPreviewViewModel`:

```kotlin
data class PhotoPreviewState(
    val activeFilterId: String,      // was index — id is stable
    val intensity: Int,              // 0..100
    val originalBitmap: Bitmap?,     // RAW unfiltered, kept in memory for re-renders
    val displayBitmap: Bitmap?,      // currently shown filtered version
    val savedUri: String,            // MediaStore Uri of filtered file
    val timestamp: String,
    val dimensions: String,
    val starred: Boolean,
)
```

Flow:

1. Screen opens with `photoId` (the filtered MediaStore Uri).
2. ViewModel decodes two bitmaps in parallel on `Dispatchers.IO`:
   - `displayBitmap` from filtered Uri (downsampled to ≤1600 px for display).
   - `originalBitmap` from `_orig` Uri **if** `saveOriginal` was true at capture time.
3. If `_orig` doesn't exist: filter swap UI hidden, intensity slider disabled. Show note: "Save originals in Settings to edit filter after capture."
4. If `_orig` exists, on filter/intensity change:
   - Debounce 200 ms.
   - Run `OffscreenLutProcessor.process(originalBitmap, newLut, newIntensity)`.
   - Update `displayBitmap` AND overwrite saved filtered file via `ContentResolver.openOutputStream(savedUri)`.
   - EXIF preserved by copying from `_orig`.

The gradient overlay (`Brush.linearGradient(listOf(activeFilter.tintTop, activeFilter.tintBottom))`) is removed entirely.

## Runtime — thumbnails

```kotlin
class ThumbnailRenderer(private val context: Context) {
    private val referenceImage: Bitmap by lazy {
        BitmapFactory.decodeStream(context.assets.open("thumbs/reference.png"))
    }
    private val cache = LruCache<String, Bitmap>(64)

    suspend fun thumbnailFor(filter: Filter): Bitmap = withContext(Dispatchers.Default) {
        cache[filter.id]?.let { return@withContext it }
        val lut = LutCache.get(context, filter)
        val out = applyLutCpuTrilinear(referenceImage, lut, intensity = 1f)
        cache.put(filter.id, out)
        out
    }
}
```

CPU trilinear LUT lookup is fine for 64×64 thumbnail size (~4K pixels). All ~30 thumbnails generated in background within ~1 s on first launch.

Wired into `FilterThumb`, `FilterRibbon`, `FilterGrid`, `HeroPreview` — replaces today's gradient + emoji placeholder.

## Error handling

| Failure | Behavior |
|---|---|
| `.cube` parse error at runtime | Returns identity LUT (existing). Add `Log.w`. Live preview shows unfiltered, capture writes unfiltered JPEG. |
| `OffscreenLutProcessor` EGL init fails | Log error. Fall back: write raw `ImageProxy` JPEG bytes to MediaStore. One-time toast: "Filter could not be applied." |
| Bitmap decode OOM in capture path | Catch; fall back to raw JPEG write. Same toast. |
| Image larger than `GL_MAX_TEXTURE_SIZE` | Downscale to fit, log warning, proceed. |
| Capture callback never fires | CameraX `onError` already handled. UI re-enables shutter. |
| User deletes `_orig` externally | `PhotoPreviewScreen` detects null; locks filter editing (same path as `saveOriginal=false`). |

## Performance targets

- LUT parse + GPU upload on filter swap: <100 ms cold, <5 ms cached.
- Thumbnail generation (64×64 CPU): <30 ms per filter; all ~30 generated in background on first launch in <1 s total.
- Capture round-trip (shutter tap → saved Uri callback) at 12MP: <1.5 s on a mid-range 2022 device. Today's path is ~700 ms; offscreen pass adds ~400–600 ms (decode + GL + encode).

## Adding a filter (developer workflow)

1. Add a row to `FilterCatalog.all` (id, category, displayName, shortCode, lutAsset path).
2. Add the matching params to `FilterRecipes.all` in `bakerMain`.
3. Run `./gradlew bakeLuts`.
4. Build and install. New filter appears in browser and ribbon.

## Open questions resolved during brainstorm

- LUT source: procedural from params (not third-party `.cube` files).
- Capture pipeline: offscreen GPU re-render with same shader (not RenderEffect, not CPU).
- Intensity: lerp final pixel against original (existing shader behavior).
- Save: filtered always; original alongside iff `saveOriginal=true` (default true).
- Migration: full taxonomy replacement; `LegacyFilterIdMap` for `defaultFilterId`.
- Param surface: compact (WB + LGG + saturation + tone curve + channel mix + optional split toning).
- Bake timing: build-time Gradle task, output committed to `assets/`.
- Thumbnails: LUT applied to procedurally-generated reference image, cached per filter.
- Preview screen: shows real saved filtered JPEG, allows filter swap via offscreen re-render of original.
